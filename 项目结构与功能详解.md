# 项目结构与功能详解

本项目为一个前端博客系统，采用现代前端技术栈（如 React、Vite 等），结构清晰，资源丰富，便于扩展和维护。以下将对项目的整体结构、各主要文件夹的作用、`public` 文件夹的前端资源含义，以及 `src` 文件夹中页面、组件、功能与样式的分离实现进行详细说明。

---

## 1. 顶层结构概览

```
├── eslint.config.js           # ESLint 配置文件
├── index.html                 # 入口 HTML 文件
├── package.json               # 项目依赖与脚本配置
├── vite.config.js             # Vite 构建工具配置
├── public/                    # 静态资源与前端素材
├── src/                       # 前端源代码（React 相关）
```

---

## 2. public 文件夹详解

`public` 文件夹用于存放所有静态资源，这些资源不会被 Webpack/Vite 处理，直接通过 URL 访问。其结构丰富，主要包括：

### 2.1 banner/
- `manifest.json`：banner 相关资源的清单文件。
- `assets/`：按日期和示例分类的 banner 静态图片、动画等素材。
  - 例如 `2023-10-01/`、`example-autumn/` 等，分别存放不同日期或主题的 banner 资源。

### 2.2 examplestyles/
- 各类前端动画、交互、页面设计的 DEMO 示例，按中文名称分类。
- 每个子文件夹通常包含：
  - `index.html`：示例页面入口
  - `style.css`：样式文件
  - `script.js`：部分示例含有 JS 交互
- 例如：
  - `404页面（css）/`：404 页面样式示例
  - `窗边风景动画（css+js）/`：窗边风景动画 DEMO
  - `卡片翻转时钟（css+js）/`：卡片翻转时钟动画
  - `注册登录界面Ⅰ（css+js）/`：注册/登录界面设计
- 这些 DEMO 既可作为博客内容展示，也可为主站功能提供素材。

### 2.3 icons/
- 存放网站所需的图标资源，如头像、导航、性别等分类。

### 2.4 imgs/
- 存放图片资源，如登录和欢迎面板相关图片。

### 2.5 live2dmodels/ & live2dsrc/
- `live2dmodels/`：存放 Live2D 虚拟形象模型文件（如粉鼠团子、灵蝶之狐等）。
- `live2dsrc/`：Live2D 相关 JS 库（如 live2dcubismcore.min.js）。
- 用于博客页面的虚拟形象展示。

## 3. src 文件夹结构与分层说明

`src` 文件夹为前端核心代码，采用分层设计，便于维护和扩展。

### 3.1 页面（pages/）
- 存放顶层页面组件，每个页面对应路由的一个视图。
- 例如：
  - `Home.jsx`：主页
  - `BlogEditPage.jsx`：博客编辑页
  - `SelfSpace.jsx`：个人空间页
  - `Welcome.jsx`：欢迎页

### 3.2 组件（components/）
- 按功能模块细分，包含通用组件、首页组件、个人空间组件等。
- 结构如下：
  - `common/`：通用组件（如头像下拉、导航栏、banner 导航等）
    - `examples/`：banner 示例组件（如 ExampleAutumn、ExampleSpring、ExampleWinter）
  - `home/`：首页相关组件
    - `maid/`：女仆组件（Maid.jsx）
  - `selfspace/`：个人空间相关组件
    - `SelfspaceProfileAccordion/`：个人空间资料手风琴组件

### 3.3 hooks/
- 自定义 React Hook，如 `useAuthState.js` 用于管理认证状态。

### 3.4 utils/
- 工具函数与脚本。
- 例如 `bannergrab/`：用于抓取和构建 banner 资源的 Node 脚本。

### 3.5 样式（styles/）
- 按模块细分的 CSS 文件，确保样式与组件解耦。
- 结构如下：
  - `common/`：通用组件样式
  - `home/`：首页及子模块样式
  - `selfspace/`：个人空间及子模块样式
  - `welcome/`：欢迎页样式

### 3.6 入口文件
- `main.jsx`：应用入口，挂载 React 根组件。
- `App.jsx`：顶层组件，负责路由与全局布局。
- `index.css`、`App.css`：全局样式。

---

## 4. 分层与解耦实现说明

- **页面（pages）**：只负责页面级布局和路由，调用下层组件。
- **组件（components）**：实现具体功能和 UI，按业务模块细分，便于复用。
- **hooks**：封装通用逻辑，提升代码复用性。
- **utils**：存放与 UI 无关的工具函数或脚本。
- **styles**：所有样式均独立于组件，按模块归类，避免样式污染。
- **public 资源**：所有静态资源（图片、动画、DEMO、模型等）均存放于 public，便于直接引用和管理。

---

## 5. 总结

本项目结构清晰，采用现代前端最佳实践：
- 静态资源与源代码分离，便于管理和部署。
- 页面、组件、功能、样式分层明确，易于扩展和维护。
- `public` 文件夹资源丰富，既服务于主站功能，也为博客内容和 DEMO 展示提供素材。
